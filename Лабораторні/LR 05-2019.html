Передача даних між HTML-формою та PHP-сценарієм

Мета роботи

Навчитися обробляти та отримувати дані з html-форми за допомогою PHP

Обладнання

Персональний комп'ютер. Пакет програм XAMPP. Текстовий редактор Sublime Text 3 або IDE NetBeans. Web-браузер Chrome, Firefox, Opera

Хід роботи

  1. Впевнитись, що пакет XAMPP встановлено та web-сервер Apache запущений
  2. Перейти до каталогу C:\xampp\htdocs\ та очистити його
  3. Створіть нову форму. Для цього визначте файл form.php, в який помістіть наступний вміст:

    <!DOCTYPE html> <html> <head> <meta charset="utf-8"> </head> <body> <h3>Вхід</h3> <form action="login.php" method="POST"> Логін: <input type="text" name="login"> Пароль: <input type="text" name="password"> <input type="submit" value="Увійти"> </form> </body> </html> </body> </html>

  4. Збережіть файл та зверніть увагу на атрибути тегу form
  5. Тепер створіть файл login.php, який буде мати такий зміст:

    <?php $login = ""; $password = ""; if(isset($_POST['login'])) $login = $_POST['login']; if (isset($_POST['password'])) $password = $_POST['password']; echo "Ваш логін: $login <br> Ваш пароль: $password"; ?>

    6. Ми можемо звернутися до форми і після натискання кнопки введені дані методом POST будуть відправлені скрипту login.php.

  6. Необов'язково відправляти дані форми іншому скрипту, їх можна обробити в тому ж файлі. Для цього змініть файл form.php наступним чином:

    <!DOCTYPE html> <html> <head> <meta charset="utf-8"> </head> <body> <div> <?php if(isset($_POST['login']) && isset($_POST['password'])){ $login=$_POST['login']; $password = $_POST['password']; echo "Ваш логін: $login <br> Ваш пароль: $password"; } ?> </div> <h3>Вхід на сайт</h3> <form method="POST"> Логін: <input type="text" name="login"> Пароль: <input type="text" name="password"> <input type="submit" value="Увійти"> </form> </body> </html>

    Збережіть файл та оновіть вкладку у браузері. Впевніться, що форма коректно обробляє дані, що були відправлені методом POST.

  7. Виконати індивідуальне завдання

    1. Створити форму зворотнього зв'язку, де вказати ім'я, почту та текст повідомлення.
    2. Для попередньої задачі створити механізм перевірки правильності вводу даних.
  8. Впевнится, що всі вихідні HTML-сторінки є валідними, використавши валідатор HTML-коду https://validator.w3.org/. За необхідності, виправити помилки та зауваження
